🧠 AI 开发核心概念图解

Vibe Coding · Agent · SKILL · MCP · CLI · Tools — 一页搞懂

🎸

Vibe Coding（氛围编程）

用自然语言描述意图，AI 帮你写代码。你负责"调方向"，AI 负责"写代码和改 Bug"。

💡 例：你说"帮我写一个读取飞书文档的 Python 脚本"，AI 直接生成代码。

理念 / 新范式
🤖

Agent（智能体）

能自主规划并执行任务的 AI。理解目标 → 拆解步骤 → 调用工具 → 检查修正，全程无需你一步步指挥。

💡 例：你说"帮我把 GitHub 上的代码部署到服务器"，Agent 自己：拉代码 → 检查依赖 → 打包 → 部署。

执行者
🧩

SKILL（技能）

预封装的"专业能力包"。装了 pptx 技能，Agent 就知道做 PPT 的最佳实践；用哪个装哪个。

💡 例：装了 lark-doc 技能后，Agent 就知道如何正确调用飞书文档 API 来创建和编辑文档。

专业知识
🔌

MCP（Model Context Protocol）

AI 与外部系统之间的"标准接口协议"。就像 USB 接口，让 AI 能标准化地对接各种外部服务。

💡 例：通过飞书 MCP，AI 能直接帮你发消息、读文档、建日程；通过 GitHub MCP，能创建 PR、查看 Issue。

连接协议
⌨️

CLI（命令行界面）

通过文字命令操作工具的方式，没有图形界面。AI Agent 可以代替你敲命令，你只需说意图。

💡 例：你说"部署项目"，Agent 在后台执行 git push、docker build、npm install 等命令。

交互方式
🔧

Tools（工具）

Agent 可以调用的"具体能力单元"：读文件、写文件、搜索代码、执行命令……没有 Tools，Agent 只能说不能做。

💡 例：Read 工具读取本地配置文件，WebSearch 搜索 API 文档，RunCommand 执行数据库迁移脚本。

执行手段

🔄 Agent 工作流程

sequenceDiagram
    participant U as 👤 用户
    participant A as 🤖 Agent
    participant S as 🧩 SKILL
    participant T as 🔧 Tools
    participant M as 🔌 MCP

    U->>A: 用自然语言描述需求
    A->>A: 理解目标 & 规划步骤
    A->>S: 加载相关 SKILL（获取专业知识）
    S-->>A: 返回最佳实践 / 操作手册
    A->>T: 调用 Tools（读文件 / 搜索 / 写代码）
    T-->>A: 返回执行结果
    A->>M: 需要外部数据时，通过 MCP 连接
    M-->>A: 返回外部系统数据
    A->>A: 检查结果 & 自我修正
    A->>T: 调用 Tools 生成最终产物
    T-->>A: 产出文件 / 代码 / 报告
    A-->>U: ✅ 交付结果
    U->>A: 反馈 & 迭代（可选）
