🎸 Vibe Coding 指南

用自然语言驱动开发，让 AI 成为你的编程搭档

解决什么问题？

🚫 问题 1：编程门槛太高

传统编程需要学习语法、框架、工具链，从零到一至少需要几个月。很多人有好的想法，但被"不会写代码"挡在门外。

🚫 问题 2：重复造轮子

即使会写代码，很多功能（登录、支付、CRUD）都是重复的。开发者把时间花在"写已经有人写过的东西"上。

🚫 问题 3：需求到实现的鸿沟

产品经理说"做一个用户中心"，开发需要理解需求、设计架构、写代码、测试……中间有大量翻译损耗。

✅ Vibe Coding 的解决方案

用自然语言直接描述意图，AI 理解并实现。你只需要"调方向"，不需要"写代码"。

什么是 Vibe Coding？

Vibe Coding（氛围编程）是由 Andrej Karpathy 提出的新概念。核心思想是：你只需要描述"想要什么"，AI 负责"怎么实现"。

你不再是逐行写代码，而是像产品经理一样"调方向"，AI 像工程师一样"写代码和改 Bug"。

💡 核心理念

人负责"意图"，AI 负责"实现"。你只需要知道"要什么"，不需要知道"怎么写"。

Trae 实战教程

1. 基础对话模式

打开 Trae 后，直接在对话框输入你的需求：

Trae 对话框示例
# 直接描述需求 帮我写一个 React 组件，显示用户列表，支持搜索和分页 # 描述问题 这个按钮点击后没有反应，帮我看看哪里有问题 # 描述重构 把这个 class 组件改成 hooks 写法

2. 使用 @ 引用上下文

使用 @ 符号可以引用特定文件或内容，让 AI 更精准理解上下文：

@ 引用语法
# 引用文件 @src/components/UserList.tsx 帮我优化这个组件的性能 # 引用整个文件夹 @src/api/ 帮我理解这些 API 的调用关系 # 引用选中的代码 （选中代码后）这段代码有性能问题吗？ # 引用终端输出 @终端 帮我分析这个报错

3. 使用 / 命令

斜杠命令可以快速执行特定操作：

/ 斜杠命令
# 代码解释 /解释 这个函数是干什么的 # 代码优化 /优化 这段代码的性能 # 生成测试 /test 为这个函数生成单元测试 # 代码审查 /review 审查这段代码的质量

4. Tab 代码补全

在编辑代码时，Trae 会自动预测你想要写的代码：

Tab 补全示例
// 输入注释，等待补全 // 计算两个日期之间的工作日数量 // 按 Tab 接受 AI 的建议 // 输入函数名，自动补全参数 function handleSubmit( // AI 会自动补全参数类型和函数体
💡 提示

Tab 补全的准确率取决于你的注释和上下文描述是否清晰。好的注释 = 好的补全。

5. 代码段筛选

当 AI 生成多段代码时，你可以选择性采纳：

  • 接受全部：点击"Accept All"或按 Cmd+A
  • 逐个接受：点击单个代码块的"Accept"
  • 拒绝：点击"Reject"或按 Esc
  • 部分接受：选中需要的部分，只接受选中内容
Vibe Coding 最佳实践

✅ Do（推荐做法）

  • 先描述整体，再细化局部：先说"我要做一个电商网站"，再说"先实现商品列表页"
  • 提供参考示例：给 AI 一个你喜欢的网站链接，说"参考这个风格"
  • 分步验证：每完成一步就测试，不要等全部写完
  • 保留对话记录：方便回溯和迭代

❌ Don't（避免做法）

  • 不要一次性提太多需求：会让 AI 混乱
  • 不要盲目接受所有代码：AI 也会犯错
  • 不要跳过测试：Vibe Coding 不等于不测试
  • 不要忽略版本控制：随时 commit，方便回滚
⚠️ 注意

Vibe Coding 适合快速原型和中小项目。大型项目仍需要架构设计和代码审查。
