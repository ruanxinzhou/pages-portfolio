⌨️ CLI 设计指南

命令行交互 - AI Agent 与系统的沟通方式

什么是 CLI？

CLI（Command Line Interface）是命令行界面，通过文字命令操作计算机系统。

在 AI Agent 场景中，CLI 是 Agent 与操作系统交互的主要方式：

  • 执行命令：运行脚本、构建项目
  • 文件操作：创建、读取、修改文件
  • 系统管理：进程管理、服务启停
  • 版本控制：Git 操作
💡 核心价值

CLI 让 AI 可以像人类开发者一样操作电脑，而且速度更快、不会犯错。

Agent 常用 CLI 命令

1. 文件操作

# 查看文件 ls -la # 列出所有文件 cat file.txt # 查看文件内容 head -n 20 file.txt # 查看前 20 行 tail -f log.txt # 实时查看日志 # 创建/修改文件 touch newfile.txt # 创建空文件 mkdir -p dir/subdir # 创建目录 cp file.txt backup.txt # 复制文件 mv old.txt new.txt # 重命名 rm file.txt # 删除文件

2. Git 操作

# 查看状态 git status # 查看工作区状态 git diff # 查看修改内容 git log --oneline -10 # 查看最近 10 条提交 # 提交代码 git add . # 暂存所有修改 git commit -m "feat: ..." # 提交 git push # 推送到远程 # 分支操作 git branch feature-x # 创建分支 git checkout feature-x # 切换分支 git merge feature-x # 合并分支

3. 项目管理

# Node.js 项目 npm install # 安装依赖 npm run build # 构建项目 npm test # 运行测试 npm run lint # 代码检查 # Python 项目 pip install -r requirements.txt python -m pytest # 运行测试 black . # 代码格式化
CLI 使用最佳实践

✅ 安全操作

  • 操作前检查：执行删除前先确认文件存在
  • 使用 dry-run：支持预览模式的命令先预览
  • 备份重要文件：修改前先备份
  • 避免 root 权限：除非必要，不使用 sudo

✅ 错误处理

  • 检查退出码：每个命令后检查 $?
  • 捕获错误输出：使用 2>&1 合并输出
  • 设置超时：长命令设置超时时间
  • 记录日志：重要操作记录到日志文件
⚠️ 危险命令

以下命令需要特别小心，AI Agent 应该避免或谨慎使用：

# 高危命令（Agent 应避免） rm -rf / # 删除根目录 chmod -R 777 / # 修改所有权限 git push --force # 强制推送 git reset --hard # 硬重置 DROP DATABASE # 删除数据库
Agent 自动化脚本示例
#!/bin/bash # 自动化开发流程脚本 set -e # 遇到错误立即退出 echo "🚀 开始自动化流程..." # 1. 检查代码状态 echo "📋 检查代码状态..." git status # 2. 运行类型检查 echo "🔍 运行类型检查..." pnpm typecheck # 3. 运行 lint echo "🧹 运行代码检查..." pnpm lint # 4. 运行测试 echo "🧪 运行测试..." pnpm test # 5. 构建项目 echo "📦 构建项目..." pnpm build echo "✅ 所有检查通过！"
