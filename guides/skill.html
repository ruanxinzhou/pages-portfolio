🧩 SKILL 设计指南

设计可复用的 AI 技能包，赋予 Agent 专业能力

解决什么问题？

🚫 问题 1：知识流失（Knowledge Loss）

每次对话都是独立的，AI 不会"记住"上次的最佳实践。你上次告诉它"用 hooks 写 React"，下次它可能又用 class 组件。

表现：同样的问题需要反复说明；AI 的输出质量不稳定；团队成员用 AI 产生不一致的结果。

🚫 问题 2：质量不一致

不同的人用 AI，产出质量差异很大。有人写的 prompt 好，AI 就表现好；有人写得差，AI 就表现差。

🚫 问题 3：重复解释

每次让 AI 做类似任务，都要重新解释"应该怎么做"。比如"做代码审查"，每次都要说"检查安全性、性能、可读性……"

🚫 问题 4：缺乏标准化

团队中每个人用 AI 的方式不同，没有统一的流程和标准，导致输出难以整合。

✅ SKILL 的解决方案

SKILL 将最佳实践封装成可复用的"能力包"。装了 SKILL，Agent 就知道"应该怎么做"，不需要你每次重复解释。

什么是 SKILL？

SKILL 是预封装的"专业能力包"，它告诉 Agent：

  • 做什么：具体的任务目标
  • 怎么做：操作步骤和最佳实践
  • 用什么：需要的工具和资源
  • 产出什么：预期的输出格式
💡 类比

如果 Agent 是"员工"，SKILL 就是"岗位培训手册"。员工可以换，但手册提供了标准化的操作规范。

SKILL 标准结构
# SKILL 文件标准结构 ## 1. 元数据 name: skill-name version: 1.0.0 description: 一句话描述这个技能 author: 作者 ## 2. 触发条件 什么时候应该使用这个 Skill： - 当用户要求...时 - 当检测到...时 - 当任务类型为...时 ## 3. 前置条件 使用这个 Skill 前需要满足： - 条件 1 - 条件 2 ## 4. 操作流程 步骤 1: [具体操作] 步骤 2: [具体操作] 步骤 3: [具体操作] ## 5. 输入输出 输入：[接受什么输入] 输出：[产出什么结果] ## 6. 质量标准 - 标准 1 - 标准 2 ## 7. 示例 [具体的使用示例]
实际案例：代码审查 SKILL
# code-review SKILL ## 元数据 name: code-review version: 1.0.0 description: 对代码进行全面质量审查 ## 触发条件 - 用户要求代码审查 - 提交 PR 时自动触发 - Gate Check 4 阶段 ## 操作流程 阶段 1：规范符合审查 1. 检查文件结构是否符合项目规范 2. 检查命名是否遵循驼峰/下划线规范 3. 检查是否使用了禁止的 API 4. 检查内联样式（零容忍） 阶段 2：代码质量审查 1. 检查函数复杂度（圈复杂度 < 10） 2. 检查重复代码（DRY 原则） 3. 检查错误处理完整性 4. 检查类型安全（TypeScript strict） ## 输出格式 { "pass": true/false, "issues": [ { "severity": "error/warning/info", "file": "path/to/file.tsx", "line": 42, "message": "问题描述", "suggestion": "修复建议" } ], "score": 85 }
SKILL 设计原则

1. 单一职责

每个 SKILL 只做一件事，做到极致：

# ✅ 好的设计 code-review: 代码审查 test-generator: 测试生成 doc-writer: 文档编写 # ❌ 不好的设计 code-helper: 代码审查 + 测试生成 + 文档编写

2. 可组合性

SKILL 之间可以组合使用：

# 工作流可以组合多个 SKILL workflow: feature-development steps: - skill: requirement-clarification - skill: solution-design - skill: code-development - skill: code-review - skill: test-generation

3. 幂等性

相同的输入应该产生相同的输出，可以安全地重复执行。

4. 可测试性

每个 SKILL 都应该有对应的测试用例。

SKILL 调用方式

方式 1：在 Agent 定义中引用

# Agent 配置文件 name: code-developer skills: - code-development - template-audit - qa-review

方式 2：在对话中动态加载

# 用户对话 用户: 帮我审查这段代码 Agent: 我将加载 code-review 技能来进行审查... # Agent 内部 1. 识别任务类型 → 代码审查 2. 加载 code-review SKILL 3. 按照 SKILL 定义的流程执行 4. 输出标准化结果

方式 3：通过命令触发

# 使用斜杠命令 /review src/components/Button.tsx # Agent 自动加载 review SKILL
